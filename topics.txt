Tracking Files Lifecycle
• Create a file (app.txt)
• Check repository status
• Stage file using git add
• Commit changes using git commit
• Modify file and view differences using git diff
• Commit updated changes
Create dev branch, Switch branches, Make changes
in dev, Switch back to main, Compare branches.

A.1
1. Initialize Repo & Create File
git init
echo "Hello World" > app.txt

🔹 2. Check Repository Status
git status

🔹 3. Stage File
git add app.txt

🔹 4. Commit File
git commit -m "Initial commit with app.txt"

🔹 5. Modify File & Check Differences
echo "New line added" >> app.txt
git diff

🔹 6. Stage & Commit Updated Changes
git add app.txt
git commit -m "Updated app.txt with new line"

🔹 7. Create & Switch to Dev Branch
git branch dev
git checkout dev

👉 Or shortcut:

git checkout -b dev
🔹 8. Make Changes in Dev Branch
echo "Changes in dev branch" >> app.txt
git add app.txt
git commit -m "Changes made in dev branch"

🔹 9. Switch Back to Main Branch
git checkout main

🔹 10. Compare Branches
git diff main..dev

👉 Optional (see commits difference):

git log main..dev

Q3
Merging and Conflict Handling
• Merge dev branch into main using git merge
• Perform fast-forward merge
• Create conflicting changes in two branches
• Resolve merge conflicts manually
• Commit resolved changes

A.3
1. Switch to Main Branch
git checkout main

🔹 2. Merge Dev into Main (Fast-Forward Merge)

👉 Happens when main has no new commits

git merge dev

✔ This will simply move main forward (no merge commit)

🔹 3. Create a Conflict Scenario

👉 Step A: Modify file in main
echo "Main branch change" > app.txt
git add app.txt
git commit -m "Change in main branch"

👉 Step B: Switch to dev and modify same line
git checkout dev
echo "Dev branch change" > app.txt
git add app.txt
git commit -m "Change in dev branch"

🔹 4. Try Merging (This Will Cause Conflict)
git checkout main
git merge dev

⚠ You’ll see a merge conflict

🔹 5. Open File & Resolve Conflict Manually

Your app.txt will look like:

<<<<<<< HEAD
Main branch change
=======
Dev branch change
>>>>>>> dev

👉 Edit it to desired final version:

Main and Dev changes combined
🔹 6. Mark Conflict as Resolved
git add app.txt

🔹 7. Commit Resolved Changes
git commit -m "Resolved merge conflict between main and dev"

Q4
Working with Remote Repositories
• Understand local vs remote repositories
• Configure remote using git remote add
• Clone repository using git clone
• Fetch updates using git fetch
• Pull changes using git pull
• Push changes using git push
Outcome: Student can synchronize local and remote repositories.

A.4
🔹 1. Understand Local vs Remote
Local repository → Your computer (where you make changes)
Remote repository → Hosted online (like GitHub) for sharing & backup

🔹 2. Add Remote Repository
git remote add origin https://github.com/username/repo.git

👉 Check remote:

git remote -v

🔹 3. Clone a Repository
git clone https://github.com/username/repo.git

✔ This downloads the repo + sets origin automatically

🔹 4. Fetch Updates (Safe Update)
git fetch origin

✔ Downloads changes but does NOT modify your files

🔹 5. Pull Changes (Fetch + Merge)
git pull origin main

✔ Updates your local branch with remote changes

🔹 6. Push Changes to Remote
git push origin main

✔ Uploads your commits to remote repo

🔹 7. First Push (Set Upstream)
git push -u origin main

✔ After this, you can just use:

git push

Q6
Undoing Changes & Recovery
• Undo unstaged changes using git checkout
• Unstage files using git reset
• Amend commits using git commit --amend
• Use git reset (soft, mixed, hard)
• Use git revert for safe undo
• Recover commits using git reflogOutcome: Student can safely undo and recover changes.

A.6
🔹 1. Undo Unstaged Changes (Discard changes in file)
git checkout -- app.txt

✔ Restores file to last committed state

🔹 2. Unstage Files (Keep changes, remove from staging)
git reset app.txt
🔹 3. Amend Last Commit (Modify message or add changes)
git add app.txt
git commit --amend -m "Updated commit message"
🔹 4. Git Reset Types
👉 Soft Reset (keep staged + working changes)
git reset --soft HEAD~1
👉 Mixed Reset (default – keep working changes only)
git reset --mixed HEAD~1
👉 Hard Reset (⚠ deletes everything after commit)
git reset --hard HEAD~1
🔹 5. Safe Undo Using Revert
git revert HEAD

✔ Creates a new commit that undoes previous changes
✔ Safe for shared/remote repositories

🔹 6. Recover Lost Commits (Reflog)
git reflog

👉 Find commit hash, then restore:

git reset --hard <commit-hash>

Q.9
Git Hooks for Automation
• Understand client-side hooks
• Create pre-commit hook to run lint checks
• Create pre-push hook to run tests
• Automate validation before commits
• Explore post-merge hook usage

·   Initialize a Git repo, Create a file, Check status, Stage file, Commit changes, Modify file, View difference, Commit again

A.9
🔹 PART 1: Git Hooks for Automation
✅ What are Git Hooks?
Scripts that run automatically on Git events
Located in:
.git/hooks/
Types:
Client-side hooks → run on your machine (pre-commit, pre-push, etc.)
🔹 1. Create Pre-Commit Hook (Run Lint Check)
cd .git/hooks
touch pre-commit
chmod +x pre-commit

👉 Edit pre-commit file:

#!/bin/sh
echo "Running lint check..."
npm run lint

if [ $? -ne 0 ]; then
  echo "Lint failed. Commit aborted."
  exit 1
fi

✔ Prevents commit if lint fails

🔹 2. Create Pre-Push Hook (Run Tests)
touch pre-push
chmod +x pre-push

👉 Edit pre-push:

#!/bin/sh
echo "Running tests..."
npm test

if [ $? -ne 0 ]; then
  echo "Tests failed. Push aborted."
  exit 1
fi

✔ Prevents pushing broken code

🔹 3. Automate Validation Before Commits
Lint checks
Code formatting
Unit tests
✔ Ensures clean & stable code
🔹 4. Post-Merge Hook (After Pull/Merge)
touch post-merge
chmod +x post-merge

👉 Example:

#!/bin/sh
echo "Installing dependencies after merge..."
npm install

✔ Useful for:

Installing dependencies
Clearing cache
Rebuilding project
🔹 PART 2: Basic Git Lifecycle Commands
🔹 1. Initialize Repository
git init

echo "Hello Git" > app.txt
🔹 3. Check Status

🔹 4. Stage File

🔹 5. Commit Changes

🔹 6. Modify File

🔹 7. View Differences

🔹 8. Commit Again

✅ 1. Create Dev Branch
git branch dev

git checkout dev

👉 Shortcut:

git checkout -b dev
✅ 3. Make Changes in Dev
echo "Work in dev branch" >> app.txt
git add app.txt
git commit -m "Changes in dev branch"
✅ 4. Switch Back to Main
git checkout main
✅ 5. Compare Branches

👉 File differences:

git diff main..dev

👉 Commit differences:

git log main..dev
🎯 Isolation Concept
Changes in dev do NOT affect main
Enables parallel development
🔹 PART 2: Branch Deletion & Cleanup
✅ 1. Delete Local Branch
git branch -d dev

👉 Force delete (if not merged):

git branch -D dev
✅ 2. Delete Remote Branch
git push origin --delete dev
✅ 3. Remove Merged Branches
git branch --merged

👉 Delete all merged (except current):

git branch --merged | grep -v "\*" | xargs -n 1 git branch -d
✅ 4. Clean Unused Remote Branch References
git fetch --prune
✅ 5. Verify Branch List

👉 Local branches:

git branch

👉 Remote branches:

git branch -r

👉 All branches:

git branch -a
